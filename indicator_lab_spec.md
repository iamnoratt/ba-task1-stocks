# Indicator Lab Spec

## 1. 任务目标

本任务的目标不是“直接让 AI 随便做几个指标图”，而是用白盒方式完成一次可追溯的指标实验。

所谓白盒，意思是：

1. 先保留原始数据
2. 明确每个指标怎么计算
3. 明确代码每一步在做什么
4. 明确图表是由哪些字段画出来的
5. 后续可以重复执行、修改参数、比较结果

最终希望得到 4 类成果：

- 原始数据文件
- 指标分析 Notebook
- 指标结果图表
- 一个可交互的 HTML 小工具设计草案

## 2. 白盒原则

这次任务必须遵守下面几条原则：

- 不接受只有结果、没有过程
- 不接受只给一段大代码、却说不清每一步逻辑
- 不接受“AI 说是这样算的”，必须能定位到公式和字段
- 不接受只看图不看原始数据
- 不接受参数写死却没有说明

## 5. 指标范围

第一版建议先做 4 类指标，不要一开始做太多：

1. 移动平均线（MA）
2. MACD
3. 布林带（Bollinger Bands）
4. 振幅或波动相关指标

如果时间够，再补：

5. RSI

## 6. 指标定义

### 6.1 移动平均线

建议至少计算：

- MA5
- MA10
- MA20

用途：

- 看短期与中期趋势
- 为后面判断价格是否站上均线提供依据

### 6.2 MACD

参数建议：

- EMA short = 12
- EMA long = 26
- Signal = 9

输出字段建议：

- DIF
- DEA
- MACD_Hist

用途：

- 看趋势强弱
- 看红绿柱变化
- 看是否存在动量反转迹象

### 6.3 布林带

参数建议：

- Window = 20
- Std multiplier = 2

输出字段建议：

- BB_Middle
- BB_Upper
- BB_Lower

用途：

- 判断价格是否接近或突破上轨/下轨
- 观察当前价格是否处于相对极端位置

### 6.4 振幅指标

建议计算：

- Daily_Amplitude = (High - Low) / Prev_Close

用途：

- 看单日波动是否明显放大
- 和趋势指标配合使用

### 6.5 RSI（可选增强）

参数建议：

- RSI14

用途：

- 辅助判断超买超卖
- 但不能机械地把高 RSI 理解成“必跌”

## 7. 处理流程

Notebook 必须按这个顺序展开：

### Step 1. 读取原始数据

要做的事：

- 读取 CSV
- 展示前 5 行
- 检查列名
- 检查日期是否升序

必须回答的问题：

- 原始数据有哪些字段？
- 有没有缺失值？
- 日期范围对不对？

### Step 2. 做基础清洗

要做的事：

- 转换日期格式
- 确保价格字段为数值型
- 确保成交量字段为数值型

必须回答的问题：

- 哪些字段参与指标计算？
- 是否有异常值需要处理？

### Step 3. 逐个计算指标

这一部分不要糊成一团，必须分块：

1. 先算 MA
2. 再算 MACD
3. 再算布林带
4. 再算振幅
5. 最后统一检查空值

必须回答的问题：

- 哪些指标会在前几行出现空值？
- 为什么会有空值？
- 为什么这些空值不能直接填 0？

### Step 4. 生成图表

建议至少做 4 张图：

1. 收盘价 + MA5/MA10/MA20
2. MACD 图
3. 布林带图
4. 振幅变化图

必须回答的问题：

- 每张图是由哪些字段画出来的？
- 当前图上看到的现象是什么？

### Step 5. 形成观察结论

这里不是做交易建议，而是做“客观描述”。

可写的结论类型：

- 当前价格位于均线之上还是之下
- MACD 红柱还是绿柱
- 布林带是否被突破
- 最近波动是否放大

不要写：

- 一定会涨
- 一定会跌
- 明确买入卖出建议

## 8. HTML 工具目标

老师这段话里很重要的一点，是希望把指标分析变成一个可交互的小工具。

第一版 HTML 工具建议只实现下面这些需求：

- 左侧选择股票
- 右侧输入参数
- 中间显示图表
- 调整参数后图表重新变化

建议参数：

- MA window
- MACD short/long/signal
- Bollinger window/std

第一版不一定要全部真实联动，但设计上要先说清楚：

- 这个工具给谁用
- 用户能调什么
- 调完之后会看到什么变化

## 8.1 参考老师截图后的界面要求

根据老师给的动态图截图，这个工具的界面风格和结构可以明确为：

### A. 左侧是控制面板

左栏不是一堆散乱按钮，而是分成几个“指标卡片”：

1. 股票选择卡片
2. BOLL 参数卡片
3. RSI 参数卡片
4. MACD 参数卡片

每张卡片里都应该包含：

- 标题
- 小圆点颜色标识
- 参数名称
- 滑块或输入框
- 当前参数值
- 一小块关键结果摘要

这意味着第一版 HTML 工具不能只放一个下拉框和一张图，而要做成“左侧控制，右侧结果”的布局。

### B. 右侧是多图联动区

老师截图里右侧不是单图，而是分成了 3 个主要图层：

1. 价格走势 + 布林带 + 多条均线
2. RSI 图
3. MACD 图

因此第一版至少要实现这种纵向结构：

- 顶部主图：价格与布林带
- 中部副图：RSI
- 底部副图：MACD

### C. 顶部有时间范围切换

截图右上角有：

- 全部
- 1月
- 3月
- 6月
- 1年

所以建议你在设计里保留一个“时间窗口切换区”，即使第一版还没完全联动，也应该在界面上预留出来。

### D. 主图上方要有图例

截图里主图不是直接画线，而是把这些对象通过图例明确出来：

- 上轨
- 下轨
- 中轨
- MA5
- MA10
- MA20
- MA60
- 收盘价

这说明你的图不是“只画一条价格线”就够了，而要让用户明确知道每条线代表什么。

### E. 参数卡片要显示即时结果

截图里左侧 BOLL 卡片不只是调参数，还会显示：

- 上轨
- 中轨
- 下轨
- 带宽

RSI 卡片还会显示：

- 最新值
- 状态标签，例如“超卖”

这说明工具不只是“参数输入器”，还应该是“当前状态摘要器”。

## 8.2 第一版 HTML 工具的最低还原目标

如果你想尽量贴近老师截图，但又不把任务做得太大，第一版建议做到下面这些：

### 第一层：布局还原

先还原界面结构：

- 左栏固定宽度控制卡片
- 右栏三段图表区域
- 顶部时间切换按钮

### 第二层：静态参数展示

先让界面能够显示：

- 当前股票名
- 当前收盘价
- 当前涨跌幅
- 当前日期范围
- 当前 BOLL、RSI、MACD 参数

### 第三层：基础联动

至少实现：

- 切换股票时，图变
- 调整 BOLL 参数时，主图变
- 调整 RSI 周期时，RSI 图变
- 调整 MACD 参数时，MACD 图变

### 第四层：状态摘要

建议自动显示这些文本结果：

- 当前价格位于中轨上方还是下方
- 当前是否突破布林带上轨/下轨
- RSI 当前值以及是否超买/超卖
- MACD 当前红柱/绿柱状态

## 8.3 图表库建议

老师也提到过 `Chart.js` 的思路，所以本任务前端建议优先考虑：

- `Chart.js`

原因：

- 比较容易上手
- 做折线图和多图布局相对直接
- 适合本地 HTML 工具
- 后续可以离线使用

如果后面你想进一步增强，可以考虑：

- `ECharts`

但第一版不建议一开始就换复杂方案。

## 8.4 你应该从截图里学到什么

这张图给你的最重要启发不是“做得好看”，而是：

1. 指标工具应该把“参数输入”和“结果观察”放在一起。
2. 主图和副图要分层，不要把所有指标堆在一张图上。
3. 关键状态要直接展示，不要让用户自己盯着线去猜。
4. 工具要支持改参数和换股票，这样才有实验价值。
